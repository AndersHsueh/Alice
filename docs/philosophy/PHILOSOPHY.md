# Alice 设计哲学

> 本文件是所有设计决策的判断标准。每当面临技术选型、功能取舍、架构决策时，先对照这四条问一句"这符合吗？"——不符合的，哪怕技术上更优雅，也不做。

---

## 内部定位（一句话）

> **本地优先、岗位无关、Skills 可叠加、人机可接力、有环境感知和社会记忆的工作伙伴系统。**

这句话不放 README，它是内部判断锚点。

面向用户的说法是：

> **"我不代替岗位，我让每个岗位都有自己的 Claude Code。"**

---

## 一、我不代替岗位，我让每个岗位都有自己的 Claude Code

Alice 不站在岗位的对立面。不替代会计、HR、PM——而是让他们各自拥有一个懂自己工作流的工具。

现有的 AI 编程工具（claude-code、cursor、copilot CLI）全部瞄准程序员。这是行业的集体路径依赖：程序员能容忍命令行，看得懂错误，不骂街。但真正能让 Agent 改变工作方式的场景，是让一个项目经理从每天 4 小时的文档/邮件/Kanban 里解放出来——这是 300% 的效率提升，不是 30%。

**Alice 瞄准的是企业中所有非程序员岗位**：项目经理、行政、HR、会计、产品、商务等。

**判断标准**：每一个功能设计，都要问"这对一个不会写代码的会计有用吗？"

---

## 二、Agent 能接住用户的工作（接力棒，不是任务交付）

绝大多数 Agent 的范式是"我来替你做完，做不了就报错"。Alice 的范式是：

> **我做不了的，你做；你做完了，我能从那个状态继续。**

工程后果：**Alice 的每一步必须把当前状态写进可读文件**（Markdown / JSON / 文件名约定），不能只在内存里。用户随时可以接管、修改、推回去。这和"一个会话从头干到尾"是完全不同的范式。

**判断标准**：如果一个功能的中间状态只存在于内存或上下文窗口里，用户接管后就丢失了，这个设计就违反了接力棒原则，必须改。

**红线**：任务失败不等于报错退出。在 Alice 里，"失败"= 把当前进度写到文件，交还给用户继续。

---

## 三、岗位是 Skills，不是写死的 Role

中国中小企业是草台班子：项目经理周一谈客户、周二写代码、周三查数据库、周四陪老板吃饭。岗位不是边界，**岗位是临时披在身上的一层皮**。

因此：

- 岗位**不能写死在 Agent 里**
- 必须用 **Skills**（可叠加、可热替换、可降级）
- 一个用户可以同时披"PM 皮 + SRE 皮 + 谈判皮"
- Skills 由用户和社区贡献，不由 Alice 内核决定

**判断标准**：每当想给某个职能写一个"专属模式"时，先想想能不能用 Skills 组合出来。能的话，就不要写死。

**红线**：只要 Alice 说出一句"这不是你这个岗位该做的事"，它就死了。

---

## 四、云端大脑 + 本地联想（双模型架构）

这是 Alice 的真正护城河。

**云端模型**（Anthropic / 第三方 API）：处理关键决策、复杂推理、代码生成。

**本地模型**（LMStudio 兼容层）：处理"联想"——持续后台扫描工作环境，维护人物关系图谱、事件时间线，在用户写邮件/文档时主动补充被忽略的上下文。

联想能力分两类：

**联想 A（环境感知）**：对会话外部环境的持续监控。token 消耗速率、网络延迟、文件系统变化、时间线提示——在合适时机插入提示到对话流。这件事工程上完全可解，且现有所有 CLI Agent 都没做好。**优先级高。**

**联想 B（社会记忆）**：持久化的人物关系图谱 + 时间维度的事件记录。写邮件时主动检索这个图谱，提示"张主任刚升职，抄送也要到他"。这些数据用户私有，不能上云，必须本地模型处理。

为什么必须双架构：

- **断网场景**（飞机、地下室、客户现场）
- **隐私场景**（人事档案、薪资、客户名单不能上云）
- **成本场景**（联想类高频低质任务，本地模型能省 80% token）

这三个场景恰好是企业 IT / 行政 / 财务最关心的。程序员可以接受全云端，企业用户接受不了。

**判断标准**：高频低质的联想类任务走本地模型；低频高质的决策类任务走云端。不要把本地模型用来做推理，也不要把云端 token 浪费在简单扫描上。

## 已作废的设计方向

记录在这里，防止将来重走弯路：

- ❌ **Role 四元组抽象**（workspace_schema / toolset / workflows / greeting_strategy）——假设岗位边界清晰，不适合中国语境
- ❌ **为每个岗位写一个独立模式**——应统一在 Skills 框架下叠加
- ❌ **自研 TUI**——壳可以借，Agent 主流程是灵魂。TUI 先复用 qwen-code，不在界面上消耗项目热情
