`20260409` `Yu`：**完成 Task #6 — QueryEngine 流式输出通过 Unix Socket 直接推送，Python 语法 + TS 构建均通过**。

### 核心改动：StreamContext 直写 Socket

**问题**：原设计中 StreamContext 将事件放入队列，由 `stream/events` 轮询拉取，延迟高且架构复杂。

**解决方案**：StreamContext 持有 `StreamWriter` 引用，事件立即写入 Socket：
```
Python QueryEngine 事件
    → StreamContext.push() / close()
    → JSON-RPC notification 帧
    → Unix Socket
    → TS AsyncGenerator
```

### 文件改动

**oh-flow/oh_flow/bridge/socket_server.py**：
1. `StreamContext` 新增 `_writer: asyncio.StreamWriter` 字段 + `set_writer()` 方法
2. `StreamContext.push()` 直接写入 `stream/event` JSON-RPC notification：
   ```python
   {"jsonrpc":"2.0","method":"stream/event","params":{"stream_id":"...","data":{...}}}
   ```
3. `StreamContext.close()` 直接写入 `stream/end` JSON-RPC notification
4. `SocketServer._current_writer`：跨消息调度传递 writer 引用
5. `_handle_client`：设置/清除 `self._current_writer`
6. `_rpc_session_stream`：创建 stream 后立即 `set_writer(_current_writer)`

**事件字段名与 TS 类型完全对齐**：
