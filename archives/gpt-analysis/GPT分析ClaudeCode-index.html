Claude Code Agent Study

Claude Code Agent 架构图 + 运行流程图 + ALICE / VERONICA 对照分析

这份页面把 Claude Code 的 Agent 设计拆成「定义层、执行层、生命周期层、特殊模式」四块， 并映射到咱们的 ALICE / VERONICA 架构里，方便快速理解：谁负责编排，谁负责执行，谁负责会话与常驻。

1. Claude Code Agent 架构图

从源码看，Claude Code 不是一个单一 Agent，而是一套「定义加载 → 执行编排 → 运行时隔离 → 持久化记忆」系统。

2. 设计特点速览

设计点 Claude Code 做法
Agent 定义 统一 `AgentDefinition`，来源多样，但按优先级合并。
上下文 用 `AsyncLocalStorage` 隔离并发 Agent 的 telemetry 和 invocation 边界。
模型策略 默认 `inherit`，并保持父模型 tier / Bedrock region 语义。
Fork 通过固定 placeholder 和 rendered system prompt 实现 prompt cache 稳定。
Memory 按 user / project / local scope 持久化，并支持 snapshot 同步。
生命周期 Spawn / Resume / Summary / Worktree cleanup 一体化。
你可以把 Claude Code 理解成：Agent 是一个可恢复、可分叉、可隔离、可缓存的任务实体，而不是一个单纯的“模型调用函数”。

4. 与 ALICE / VERONICA 的对照分析

维度 Claude Code 咱们的 ALICE / VERONICA
系统形态 Agent 体系更像“编排框架 + 任务实体” ALICE + VERONICA 是“CLI + 常驻 daemon”的双进程产品
运行核心 Agent 运行态强调 spawn / resume / background / fork VERONICA 负责会话、工具、LLM、事件流与常驻编排
前端入口 CLI / 交互 / 任务触发一体 ALICE 主要承担用户交互、TUI、命令入口
隔离方式 AsyncLocalStorage + worktree + remote + fork 可以借鉴为：会话隔离、工具隔离、任务上下文隔离
记忆系统 Agent scoped memory + snapshot sync 可映射到 profile / workspace / project 级别知识层
产品定位 更偏“Agent runtime 平台” 更偏“办公 CLI 产品 + daemon 执行中枢”

5. 结构上的启发

Claude Code 的关键启发是：把 Agent 当成一个可治理的运行单元，围绕它建立定义、上下文、权限、工作区、记忆和恢复机制。
可借鉴点 对 ALICE / VERONICA 的启发
Agent 定义中心化 把“能力 / 工具 / 模型 / memory / isolation”收敛到统一定义层。
上下文隔离 用会话级或任务级上下文，避免后台并发互相覆盖。
可恢复生命周期 让中断后的任务能依据 transcript / metadata 续跑。
特殊模式显式化 fork / worktree / remote 这类高阶模式应单独命名、单独约束。
